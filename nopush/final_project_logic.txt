#쿠폰
현재방식
1. 회원 테이블에서 회원 정보를 불러옴
2-1. 전체 발급 쿠폰인 경우 가장 상단의 전체회원에 체크
2-2. 필터를 통해 지급받을 회원을 체크
3. 쿠폰이름, 사용만료 날짜, 할인 타입, 타입에 따라 할인률/할인금액, 전체 발급 쿠폰 여부 체크박스 입력, 전체 발급 쿠폰(쿠폰 발급시마다 입력)


다른방식
//관리자페이지에서 회원에게 직접 지급
1. 쿠폰 이름, 할인 타입, 타입에 따라 할인률/할인금액 입력

//지급
1. 회원 테이블에서 회원 정보를 불러옴
2-1. 전체 발급 쿠폰인 경우 가장 상단의 전체회원에 체크
2-2. 필터를 통해 지급받을 회원을 체크
3. 지급할 쿠폰을 선택 후 쿠폰 지급

4-1. 전체 발급 쿠폰인 경우 쿠폰 지급 내역 테이블에
쿠폰지급번호: AI(pk)
쿠폰이름: 쿠폰의 이름 fk
쿠폰코드: uuid 앞 16자리 자동생성 입력
쿠폰지급회원: "everyone"
발급날짜: now()
만료날짜: 입력 or 지금으로부터 n일 선택

4-2. 특정 회원 발급 쿠폰인 경우 쿠폰 지급 내역 테이블에
쿠폰지급번호: AI(pk)
쿠폰이름: 쿠폰의 이름 fk
쿠폰코드: uuid 앞 16자리 자동생성 입력
쿠폰지급회원: 선택된 member_id (fk x)
발급날짜: now()
만료날짜: 입력 or 지금으로부터 n일 선택

//사용
1. 회원이 쿠폰조회시 uuid를 이용해 지급내역에서 있고 만료날짜가 지나지않았으며, 쿠폰 사용 내역에 없는 쿠폰만 보여줌
2. 쿠폰 사용시 사용 내역 테이블에 추가
3. 쿠폰 사용시 주문 테이블에는 coupon_code가 입력되야할 듯
 
?생일 쿠폰을 지급한다면? (관리자페이지x)
1. 관리자 페이지에서 생일 쿠폰을 발급정보에는 미리 입력되어있어야 함
2. 회원 로그인시 금일이 birthday인지 확인 && 쿠폰 지급 내역에 이번년도 지급 내역이 없어야함
3. 쿠폰 지급 4-2, 알림창 띄우기?
